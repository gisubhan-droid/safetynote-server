### TASK-001 — 공사현황 수정/삭제 기능 추가

| 항목 | 내용 |
|------|------|
| **분류** | 기능 추가 (FEAT) |
| **메뉴** | 공사 현황 |
| **상태** | ⏳ 대기 중 |
| **묶음 그룹** | 공사현황 관련 (TASK-001, TASK-002, TASK-003) |

#### 요구사항
- 등록된 공사건에 **수정** / **삭제** 기능 추가
- 삭제 조건: **진행 중인 작업 중 완료건이 없을 경우에만** 삭제 허용
  - 즉, 해당 공사에 연결된 작업이 하나라도 완료 상태이면 삭제 불가
  - 수정은 별도 조건 미지정 → 구현 시 확인 필요

#### ✅ 확정된 처리 방침 (2026-06-19 사용자 결정)
- **삭제 시 연결 데이터 처리**: **삭제 차단** 방식
  - 해당 공사에 연결된 작업(완료 포함 전체)이 하나라도 있으면 삭제 불가
  - 삭제 시도 시 "연결된 작업이 존재합니다. 작업을 모두 삭제 후 공사를 삭제해 주세요." 안내

#### 확인 필요 항목
- [ ] 현재 공사 상세 화면에 수정/삭제 버튼 UI 존재 여부 확인
- [ ] `constructions` 테이블 구조 및 연결된 작업 상태 조회 방법 확인
- [ ] 삭제 차단 조건 쿼리: `SELECT COUNT(*) FROM tasks WHERE construction_id = ?` → 1 이상이면 차단

4. 기대 동작: **공사 상세 화면에 그대로 머물러야 함**

### TASK-003 — 공사 등록 시 공사요청번호 자동생성 옵션 추가

| 항목 | 내용 |
|------|------|
| **분류** | 기능 추가 (FEAT) |
| **메뉴** | 공사 현황 → 공사 등록 |
| **상태** | ⏳ 대기 중 |
| **묶음 그룹** | 공사현황 관련 (TASK-001, TASK-002, TASK-003) |

#### 요구사항
공사 등록 폼의 공사요청번호 입력란에 **"요청번호없음"** 체크박스 추가

- **체크 해제 (기본)**: 공사요청번호 직접 입력 (현재 방식 유지)
- **체크 시**:
  - 공사요청번호 입력란 비활성화
  - 자동 생성된 번호로 채워짐

#### 자동 생성 번호 형식
```
LM_YY.MM.DD_##
```

| 구성 요소 | 설명 | 예시 |
|-----------|------|------|
| `LM` | 고정값 | `LM` |
| `YY.MM.DD` | 등록일 기준 **KST** 날짜 자동 입력 | `26.06.02` |
| `##` | 등록일 기준 등록 **순번** (2자리, 01부터 시작) | `01`, `02`, `03` ... |

**완성 예시**: `LM_26.06.19_01`, `LM_26.06.19_02`

#### 확인 필요 항목
- [ ] 공사 등록 폼 위치 (`app.js`) 확인
- [ ] `constructions` 테이블의 요청번호 컬럼명 확인
- [ ] 서버에서 당일 등록 순번 조회 로직 추가 필요 (KST 기준 날짜 필터)
- [ ] 순번 채번: `SELECT COUNT(*) FROM constructions WHERE request_no LIKE 'LM_YY.MM.DD_%'`
