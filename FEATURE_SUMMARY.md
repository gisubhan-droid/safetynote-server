## 1. 시스템 개요

**SafetyNOTE**는 외선·접속 공사 현장의 스마트 안전관리 시스템입니다.

| 항목 | 내용 |
|------|------|
| 시스템명 | SafetyNOTE (세이프티노트) |
| 버전 | v3.0 (20260705a) |
| 대상 현장 | 외선·접속 공사 현장 |
| 저장소 | `gisubhan-droid/safetynote-server` (GitHub, main 브랜치) |
| 운영 환경 | Synology NAS (DSM 7.x) + Node.js + SQLite |
| 접속 방법 | 웹 브라우저 또는 Android 앱 |

### 주요 목적
- 현장 작업자의 TBM(Tool Box Meeting) 실시 및 안전 확인 전산화
- 공사 진행 상황 실시간 추적 및 통계 제공
- 현장점검·안전교육·위험성평가 이력 관리
- 외선일보·접속일보 작성 및 월별 금액 집계

## 2. 접속 방법

### 2-1. 외부 접속 (스마트폰·현장)
```
URL: https://linkmax.myds.me:3443
프로토콜: HTTPS (보안 연결)
포트: 3443 (공유기 포트포워딩)
```

### 2-2. 내부 접속 (사무실 내부망)
```
URL: http://NAS-IP주소:3444
프로토콜: HTTP
포트: 3444 (내부망 전용)
```
> NAS-IP주소는 공유기 설정에서 NAS에 부여한 고정 IP를 사용합니다.

### 2-3. Android 앱
- GitHub Releases 또는 내부 배포 URL에서 APK 설치
- 앱 실행 시 자동으로 서버에 연결

### 2-4. 로그인
- 아이디: 관리자가 등록한 계정 ID
- 비밀번호: 초기 비밀번호 (최초 로그인 후 변경 권장)
- 로그인 상태는 JWT 토큰으로 유지 (자동 갱신)

## 5. 기능별 상세 설명

### 5-1. 공사현황 (constructions)

현재 진행 중인 공사를 등록하고 관리하는 핵심 메뉴입니다.

#### 주요 기능
- **공사 등록**: 공사명, 공사종류, 공사담당자, 시공 위치, 공사 기간 등록
- **공사 수정/삭제**: 등록된 공사 정보 수정 및 삭제
- **정산 상태 관리**: 정산요청 → 정산완료 상태 변경
- **시공통보금액 관리**: 공사별 시공 통보 금액 입력
- **첨부파일**: 관련 서류 첨부 (사진, PDF 등)
- **현장 위치 지도**: 공사 현장 위치를 지도에서 확인

#### 공사 상태 흐름
```
진행중 → 완료 → 정산요청 → 정산완료
```

#### 필터 및 검색
- 공사종류별 필터 (6종)
- 공사담당자별 필터
- 기간별 필터
- 텍스트 검색 (공사명)

---

### 5-2. 작업관리 (tasks)

공사에 연결된 세부 작업을 생성하고 작업자에게 배정하는 메뉴입니다.

#### 작업 상태 흐름
```
미배정 → 작업자배정 → 체크리스트완료 → TBM완료 → 작업진행중 → 작업완료 → 일지완료
                                                              ↓
                                                           취소(cancelled)
```

#### 주요 기능
- **작업 등록**: 작업명, 공사 연결, 작업팀 배정, 예정일 설정
- **작업자 배정**: 팀 또는 개인 단위 배정
- **체크리스트 연동**: 작업 전 안전 체크리스트 확인
- **TBM 연동**: TBM 완료 시 작업 상태 자동 업데이트
- **작업 완료 처리**: 실적 입력 및 완료 처리
- **일보 연동**: 외선일보·접속일보 작성 연결

#### 필터 카드 (탭)
- 전체 / 미배정 / 진행중 / 완료 / 취소

---

### 5-3. 근로자 — 내 작업 목록 (my-tasks)

`worker` 역할 사용자가 본인에게 배정된 작업만 조회하는 전용 화면입니다.

#### 상단 필터 카드 (4종)

| 카드 | 필터 조건 | 비고 |
|------|-----------|------|
| 전체 | 모든 작업 | |
| 진행중 | status ∈ {배정됨, 진행중, TBM완료, 체크리스트완료} | |
| 완료 | status = 완료 또는 일지완료 | |
| **금일예정** ⭐ | planned_date = 오늘(KST) AND status ≠ 취소 | 빨간 알림 뱃지 |

> **금일예정**: 오늘 날짜에 예정된 작업이 있을 경우 카드 우상단에 빨간 점(●)이 표시됩니다.

#### 검색 기능 ⭐ (신규)
- **위치**: 필터 카드 하단 검색 입력창
- **검색 대상**: 등록건명(공사명), 공사담당자 이름
- **동작**: 입력 즉시 필터링 (300ms 지연 후 반영, 서버 재요청 없음)
- **초기화**: 다른 페이지로 이동 시 검색어 자동 초기화

#### 작업 카드 표시 정보
- 공사명 / 작업명
- 담당자 / 작업팀
- 예정일 / 상태 배지
- TBM 실시 여부

---

### 5-4. TBM (Tool Box Meeting)

작업 시작 전 안전 미팅(TBM)을 실시하고 기록하는 기능입니다.

#### TBM 진행 순서
```
1. 작업 선택 → 2. 작업 유형 선택 → 3. 체크리스트 작성 →
4. 위험요인 확인 → 5. 참석자 서명 → 6. TBM 완료
```

#### 주요 기능
- **TBM 실시**: 작업별 TBM 등록 및 참석자 전자서명
- **TBM 이력 확인**: 동일 작업의 이전 TBM 이력 팝업 표시
- **작업 개시 버튼**: TBM 완료 후 즉시 작업 시작 처리
- **TBM 공유**: 카카오톡·문자 등으로 TBM 결과 공유 (URL 링크)
- **QR 코드**: 작업별 QR 코드 생성 및 인쇄

#### 작업 유형 (TBM 체크리스트 연동)
- 지중배선 / 관로 / 케이블 포설 / 접속 / 가공 / 철거 / 기타

---

### 5-5. 현장점검 (inspections)

현장 안전점검 실시 및 이력을 관리합니다.

#### 주요 기능
- **점검 등록**: 점검일, 점검자, 점검 현장, 점검 결과 입력
- **점검 이력 조회**: 기간별, 현장별 필터링
- **지적 사항 관리**: 불량 항목 등록 및 개선 조치 추적
- **사진 첨부**: 점검 현장 사진 업로드

#### 목록 표시 (모바일 카드형)
- 점검일 / 점검 현장명
- 공사담당자 / 작업팀
- 점검 결과 배지

---

### 5-6. 안전교육 (education)

법정 안전교육 실시 및 이력을 관리합니다.

#### 교육 종류

| 교육 구분 | 법적 근거 |
|-----------|-----------|
| 정기교육 | 산업안전보건법 제29조 |
| 채용 시 교육 | 신규 입사자 대상 |
| 작업내용 변경 시 교육 | 작업 변경 시 실시 |
| 특별교육 | 위험 작업 전 실시 |
| 관리감독자 교육 | 관리감독자 대상 |

#### 주요 기능
- 교육 실시 등록 (교육일, 교육시간, 교육 내용, 참석자)
- 교육 이력 조회 및 출력
- 교육 확인 서명

---

### 5-7. 위험성평가 (risk)

작업 전 위험성평가 실시 및 이력을 관리합니다.

#### 평가 종류

| 구분 | 실시 시기 |
|------|-----------|
| 정기 위험성평가 | 연 1회 이상 |
| 수시 위험성평가 | 작업 변경·사고 발생 시 |

#### 주요 기능
- 위험 요인 등록 (작업 단계별 위험 요인, 위험성 수준)
- 감소 대책 입력
- 담당자 서명 및 승인
- 이력 조회

---

### 5-8. 외선일보 / 접속일보

현장 작업 일지를 작성하고 월별 금액을 집계합니다.

#### 외선일보 (work-reports)
- 외선 공사 일별 작업 내역 기록
- 공종별 작업량 입력 (단가 × 물량 = 금액 자동 계산)
- **월별 합계 금액 조회**: `/work-reports/monthly-amount?year=YYYY&month=M`

#### 접속일보 (splice-reports)
- 접속 공사 일별 작업 내역 기록
- 공종별 집계 금액 입력
- **월별 합계 금액 조회**: `/splice-reports/monthly-amount?year=YYYY&month=M`

#### 월별 금액 집계
- 공시현황(stats-task) 페이지의 "총 시공물량" 카드에 자동 반영
- **표시 형식**: 접속 + 외선 합산, 100만 원 이상이면 `N.N백만` 표시

---

### 5-9. 공시현황 / 작업통계 (stats-task)

월별 작업 통계를 조회하는 관리자용 페이지입니다.

#### 상단 필터
- 연도 선택 (드롭다운)
- 월 선택 (드롭다운)
- **공사종류 필터** ⭐: 드롭다운+체크박스 (기본: 지장이설)

#### 요약 카드 (4종)

| 카드 | 표시 내용 |
|------|-----------|
| 총 작업건수 | 해당 월 전체 작업 수 |
| 완료건수 | 해당 월 완료된 작업 수 |
| **총 시공물량** ⭐ | 접속일보 + 외선일보 월별 합계 금액 (N.N백만) |
| 진행율 | 완료/전체 비율 (%) |

> **총 시공물량 카드**: 연도/월 변경 시 자동으로 최신 금액을 조회하여 표시합니다.

#### 상세 통계 테이블
- **공사종류별 완료건수**: 공사종류(필터 반영)별 완료 현황
- **팀별 완료건수**: 팀별 작업 완료 현황
- **팀별 진행중 현황**: 현재 진행 중인 팀별 작업 건수

---

### 5-10. 공사통계 (con-stats)

공사 현황을 년간·월간·주간 단위로 분석하는 통계 페이지입니다.

#### 기간 선택
```
[년간] [월간] [주간]  [2026년 ▼]  [조회]
```
- **년간**: 해당 연도 전체 공사 통계
- **월간**: 해당 연도/월 공사 통계
- **주간**: 선택 주간(월~일) 공사 통계 (이전/다음 주 이동 버튼)

#### 공사종류 필터 ⭐ (신규)
```
[공사종류 ▼ 지장이설]
  ├ [전체선택] [전체해제]
```

## 7. NAS 설치 참고 정보

### 7-1. 시스템 요구사항

| 항목 | 권장 사양 |
|------|-----------|
| 기기 | Synology NAS (DS220+, DS923+ 등) |
| OS | DSM 7.x 이상 |
| Node.js | 18.x 이상 (NAS 패키지 매니저 설치) |
| 메모리 | 4GB 이상 권장 |
| 저장소 | 10GB 이상 여유 공간 |
| 네트워크 | 공유기 포트포워딩 가능한 환경 |

### 7-2. 포트 구성

| 포트 | 프로토콜 | 용도 | 설정 위치 |
|------|----------|------|-----------|
| **3443** | HTTPS | 외부 접속 | 공유기 포트포워딩 + NAS 방화벽 |
| **3444** | HTTP | 내부망 접속 | NAS 방화벽만 |

> ※ 리버스 프록시 미사용 (이중 SSL 충돌 방지)

### 7-3. SSL 인증서

Synology NAS의 Let's Encrypt 인증서를 자동 탐지하여 HTTPS 서버 시작.

```
인증서 경로: /usr/syno/etc/certificate/_archive/<DEFAULT>/
- fullchain.pem  (인증서 체인)
- privkey.pem    (개인 키)
```

> `node-server.ts`가 시작 시 위 경로를 자동 감지하며, 인증서 없을 경우 HTTP(3444)만 시작됩니다.

### 7-4. 환경 변수

| 변수명 | 기본값 | 설명 |
|--------|--------|------|
| `PORT` | `3443` | 외부 HTTPS 포트 |
| `HTTP_PORT` | `3444` | 내부 HTTP 포트 |
| `DB_FILE` | `./safety.db` | SQLite 데이터베이스 파일 경로 |
| `JWT_SECRET` | (필수) | JWT 인증 토큰 시크릿 키 |
| `UPLOAD_DIR` | `./uploads` | 첨부파일 저장 경로 |

### 7-5. 프로세스 관리 (PM2)

```bash
# PM2 전역 설치
npm install -g pm2

# SafetyNOTE 시작
pm2 start ecosystem.config.js --name safetynote

# 상태 확인
pm2 list

# 로그 확인
pm2 logs safetynote --nostream

# 자동 시작 설정 (NAS 재부팅 시)
pm2 startup
pm2 save
```

### 7-6. 데이터베이스 설정

- **엔진**: SQLite (better-sqlite3)
- **WAL 모드**: 활성화 (동시 읽기 성능 향상)
- **foreign_keys**: ON
- **캐시 크기**: 32MB

```bash
# DB 파일 위치 (권장)
/volume1/safetynote/safety.db

# 백업 (권장: cron 일별 자동 백업)
cp safety.db safety_backup_$(date +%Y%m%d).db
```

### 7-7. GitHub Secrets (자동 배포 설정)

자동 업데이트를 위해 GitHub Actions에 다음 Secrets 등록 필요:

| Secret 이름 | 값 | 용도 |
|-------------|-----|------|
| `NAS_HOST` | NAS IP 주소 | SSH 접속 대상 |
| `NAS_USER` | NAS 사용자명 | SSH 로그인 |
| `NAS_SSH_KEY` | NAS SSH 개인키 | SSH 인증 |
| `NAS_DEPLOY_PATH` | 배포 경로 | 예: `/volume1/safetynote` |

## 10. 자주 묻는 문제 (FAQ)

### Q1. 앱 접속이 안 됩니다.
**A**: 다음 순서로 확인하세요.
1. 공유기 포트포워딩: 외부 3443 → NAS IP:3443 확인
2. NAS 방화벽: 3443, 3444 포트 허용 확인
3. PM2 상태: `pm2 list` → safetynote가 `online` 상태인지 확인
4. SSL 인증서: `/usr/syno/etc/certificate/_archive/` 경로에 인증서 파일 존재 여부

### Q2. 로그인이 안 됩니다.
**A**: 관리자에게 계정 상태 확인 요청. `is_active = 1` 여부 및 비밀번호 초기화 요청.

### Q3. TBM 서명 후 완료 처리가 안 됩니다.
**A**: 모든 참석자의 서명 완료 여부 확인. 1명 이상 서명 누락 시 완료 처리 불가.

### Q4. 공사통계에서 필터를 변경해도 반영이 안 됩니다.
**A**: 드롭다운에서 체크박스 변경 후 자동 반영됩니다. 아무것도 선택하지 않으면 전체 공사가 조회됩니다.

### Q5. 총 시공물량 카드가 0으로 표시됩니다.
**A**: 해당 월의 외선일보 또는 접속일보가 등록되지 않았을 수 있습니다. 연도/월 선택 후 조회 버튼을 눌러 새로고침하세요.

### Q6. 내 작업 목록에서 작업이 검색되지 않습니다.
**A**: 검색창에 등록건명(공사명) 또는 공사담당자 이름을 입력하세요. 검색은 부분 일치(포함)로 동작합니다.

### Q7. 금일예정 카드에 빨간 점이 표시됩니다.
**A**: 오늘 날짜에 예정된 작업이 있다는 알림입니다. 금일예정 카드를 클릭하면 오늘 예정 작업만 필터링됩니다.

### Q8. 서버 업데이트 후 기존 데이터가 사라졌습니다.
**A**: SQLite DB 파일(`safety.db`)은 업데이트 시 변경되지 않습니다. DB 파일 경로가 올바른지 확인하세요. 데이터 손실 시 백업 파일에서 복구 가능합니다.

### Q9. 앱 화면이 하얗게 표시됩니다.
**A**: 브라우저 캐시 강제 새로고침 (`Ctrl+Shift+R` 또는 `Cmd+Shift+R`)을 시도하세요. 서버 오류일 경우 `pm2 logs safetynote --nostream`으로 오류 확인.

### Q10. NAS를 새로 설치할 때 기존 데이터를 이전하려면?
**A**:
1. 기존 NAS에서 `safety.db` 파일과 `uploads/` 폴더 백업
2. 신규 NAS에 SafetyNOTE 설치
3. 백업한 `safety.db`와 `uploads/`를 신규 NAS의 배포 경로에 복사
4. PM2로 서버 재시작
